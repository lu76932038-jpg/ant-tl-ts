# 殸木社区生产环境反馈处理工作流

这个工作流旨在自动化处理“殸木社区”生产环境的反馈信息，完成数据自动读取、智能分类、排查验证、产品原型设计，并在后台建立由 AI（Antigravity）全权维护的任务进度管理体系。经管理员审批后执行开发与部署，并向社区用户同步进展。

### 1. 自动读取生产环境信息
- 由 Antigravity 定期或按需自动通过接口（API）、数据库直连或网页抓取（Browser Subagent）等方式，获取生产环境“殸木社区”的最新反馈、异常运行日志或用户工单。
- 梳理并总结提取到的所有未被处理的信息列表。

### 2. 智能分类与验证排查
- **分类研判**：针对收集到的信息，进行专业研判并将其分类为：**系统缺陷**、**系统问题**、**系统需求**。
- **缺陷验证（开发环境）**：对于系统缺陷，在开发环境中尝试复现触发条件，确认是否为真实 Bug，并提取相关报错日志。

### 3. 系统需求转化与原型设计
- 针对判定为**系统需求**的内容：
  - 使用 **Stitch** 进行需求展开，制作并输出标准的《需求文档》。
  - 使用 **Stitch** 制作需求原型，提供界面交互或功能方案的具象化展示。

### 4. 任务管理建立与社区状态同步
- **后台任务生成**：Antigravity 根据步骤2和3的结果，在系统后台或约定好的看板中自动建立对应的“开发任务记录”。
- **AI 智能跟进**：任务的“进度状态”（如：排查中、需求规划中、待审批、开发中、已发版等）完全由 Antigravity 负责维护和自动流转。
- **社区用户可见**：将当前的排查结果、预计方案或处理进度同步回调至“殸木社区”的生产环境数据库或接口，让最初提出反馈/需求的用户可以在前台直观看到跟进结果。

### 5. 管理员审批节点 (Admin Approval)
- 提取并汇总以上生成的【分类结果/验证报告】、【Stitch 原型】及【规划好的后台任务】。
- 强制校验点：**必须请求管理员（USER）进行最终审批。**
- **管理员明确点击通过或回复确认后，所分配的任务才获准正式进入开发过程。** 未经审批，禁止擅自修改代码。

### 6. 开发任务执行与自动化发版
- 在管理员审批通过后，正式启动业务逻辑的代码编写、缺陷修复或需求开发。
- 本地研发与测试完成时，调用项目的发版部署标准流程（如运行 `docs/生产环境配置/上线脚本/` 中的相关脚本）。
- **生产环境信息闭环**：上线成功后，Antigravity 不仅在本地更新状态，还必须**自动调用生产环境的相关 API 或执行数据库更新**，将生产环境后台的“任务进度”流转为【已发版/完成】，并确保该最终状态能够真实地渲染并反馈给社区内的提单用户。
